컴퓨터 언어란?

컴퓨터와 의사소통을 하기위한 언어

컴퓨터가 이해할 수 있는 언어 > binary code (이진수 명령어) 0과 1로 이루어 져있다.

이를 사람이 읽기 편하게 만들어 진 언어가 어셈블리 언어

이후 나온 고급언어 > C, python, Java


컴퓨터가 이해하기 쉬울 수록 저급언어
사람이 이해하기 쉬울 수록 고급언어

이러한 고급언어는 사람이 이해하기는 쉽지만 컴퓨터는 이해할 수 가없다.
그렇기 때문에 번역이 필요하다.

이 번역을 하는 과정을 컴파일 이라고 하고
이 번역을 하는 녀석을 컴파일러라고 한다.


Java란?

제임스 고슬링이라는 사람이 만든 언어
Write Once Run Anywhere

한번 작성하면 어디서든지 실행된다.

java 가 등장하기전 개발

C언어, C++ 언어 소스코드를 쓰려면

윈도우, 리눅스, 맥os 등 각각의 하드웨어에 따라 
운영체제의 컴파일러가 다르기 때문에 각각의 운영체제에
컴파일러를 만들어야 가능했다.
c같은 경우에는 파일이 각각 .c로 이루어져 있으나 
프로그램을 실행을 하려면 obj단위로 묶어 링커를 통해서 exe로 만들어야 프로그램 실행이 가능하다.
하드웨어인 메모리의 용량보다 큰 exe파일이면 실행이 되지 않는다

c++ 은 소멸자로 소멸을 해주어야한다.


java 소스코드는

컴파일러(바이트 코드) 를 **JVM (java virtual merchine) 자바 가상 머신**으로 인해
하나로 통일 가능케 했다.


java 의 특징에는 객체 지향 언어 이다.(클래스 로딩)
절차적인 언어에 객체 개념이 들어간 언어이다.

java는 각각의 class단위로 끊어져 있어 
jvm 메모리에 올라갈수만 있다면 class단위로 끊어서 컴파일이 가능하고
그렇기 때문에 하드웨어에 종속되지 않는다.

garbege collector 가비지 컬렉터
java는 동적메모리 관리에 따른 쓰레기값을 자동 수거 한다.
덕분에 개발자가 개발에만 집중하고 메모리 관리를 굳이 할 필요가 없어졌다.
